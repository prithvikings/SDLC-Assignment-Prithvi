# SDLC Assignment – Smart-Attend App

## Project Overview

This project is based on a university startup developing a mobile application called **Smart-Attend**. The application automatically marks student attendance when a student enters the classroom using geo-fencing technology.

Later in the project, a new requirement was introduced to integrate **Biometric Face ID verification** to prevent proxy attendance.

This assignment analyzes the limitations of the Waterfall model and explains how the Scrum framework can better handle such changing requirements.

## Why CI/CD Would Be Used

CI/CD (Continuous Integration and Continuous Deployment) would play an important role in this project.

### Continuous Integration

Developers frequently merge their code into a shared repository. Each time new code is added, automated tests run to ensure the system still works correctly. This helps detect bugs early and keeps the codebase stable.

### Continuous Deployment

Once the code passes testing, updates can automatically be deployed to the application servers. This ensures that new features and improvements reach students quickly.

### Benefits in this Project

Using CI/CD would allow:

- Faster delivery of new features like Face ID verification
- Automatic bug detection through testing pipelines
- Frequent updates without disrupting the application
- Faster feedback from users and teachers

Overall, CI/CD helps maintain reliability while allowing continuous improvements to the Smart-Attend system.
